## Context

这次变更不是新增大型插件市场，而是把已有源码 TODO 收束成可验证的最小能力：依赖检查要能阻止明显不兼容插件，本地/文件 URL 安装要能替代硬编码内置插件列表，世界观地点引用检查要能发现作者显式标记的未定义地点。

## Decisions

1. **版本比较只支持可维护的 semver 子集。** 支持 `>=`、`>`、`<=`、`<`、`=`、`^`、`~` 和裸版本，覆盖当前 manifest 中最常见的 `>=0.20.0`。不新增外部依赖，避免只为一个小比较器扩大依赖面。
2. **远程安装先落到本地路径与 `file://`。** HTTP(S) 下载需要认证、缓存、校验和供应链安全策略，本次只实现可审计的本地源解析，并对 HTTP(S) 给出明确错误。
3. **地点引用用显式标记降低误报。** 脚本检查正文中的 `@地点:名称`、`地点：名称`、`位置：名称` 等显式标记，而不是猜测所有中文专名，避免创作正文被大量误报淹没。

## Risks

- semver 子集无法覆盖复杂表达式：遇到不支持的表达式时阻止安装并说明原因。
- 地点标记需要作者/agent 配合：这是为了避免误报；普通自然语言地点仍留给人工审阅。
