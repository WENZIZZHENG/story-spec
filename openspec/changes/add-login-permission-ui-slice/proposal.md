# 登录/权限 UI 只读切片

## 背景

`apps/web/` 已有独立前端 shell，但用户还不能在首屏看见当前会话、项目角色、关键权限动作和禁用原因。多人平台已有 session/project guard 与角色权限模型；下一步需要让前端先展示这些状态，而不是直接实现完整账号注册、邀请或团队管理。

## 目标

- 在独立 Web shell 中新增登录/权限只读 UI contract。
- 展示 session 绑定状态、当前角色、可执行动作、被禁用动作和权限不足原因。
- 在架构契约中声明登录/权限 UI 的 endpoint、边界和非目标。
- 保持本切片不创建账号、不修改成员、不发邀请、不绕过 server 权限守卫。

## 非目标

- 不实现真实注册、登录、登出、邀请、角色变更或团队管理。
- 不新增 React/Vite/Next/Tailwind 等前端栈。
- 不替换本机 `storyspec app` fallback。
- 不改变后端权限判定语义。
