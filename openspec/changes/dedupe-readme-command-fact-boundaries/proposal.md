# README 高频命令去重与事实边界巡检

## 背景

README 高频命令表中 `storyspec server` 出现重复行。虽然不影响 CLI 行为，但会让公开文档看起来不够可信，也容易削弱“已实现能力”和“路线图能力”的边界。

本轮只收口公开文档事实边界，不新增产品能力。

## 目标

- 去掉 README 高频命令表中的重复命令行。
- 为 README 高频命令表增加轻量回归测试，防止同一命令重复出现。
- 保持 `storyspec app` 与 `storyspec server` 的实验性边界：不把账号、云端、完整 SaaS、真实 worker、真实数据库全量接入或富文本编辑器写成已实现能力。
- 同步 P0 路线图状态，让下一步进入完整 App 产品体验设计。

## 非目标

- 不修改 CLI、App、Server、模板或生成产物行为。
- 不承诺多人协作、云端 SaaS、富文本或真实 worker 已完成。
- 不清理历史归档中的旧阶段描述。
