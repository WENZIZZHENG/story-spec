## Why

当前 `storyspec app` 仍是实验性本机工作台，用户打开 Web 后不容易判断“从哪里开始”“当前故事卡在哪里”“哪些内容只是候选”“哪些动作会写入正式故事”。如果直接进入完整 App 前端重写，容易把不清晰的产品模型固化到路由、API 和数据模型里。

本 change 先沉淀完整 App 的产品体验设计，作为后续 API contract、前端架构和多人在线写作平台实现的输入。

## What Changes

- 确认首版产品优先级为“作者 + 团队平衡”。
- 确认信息架构采用“故事驾驶舱居中”。
- 确认首批页面范围：项目/工作区入口作为 App shell 必备入口；故事内主页面为故事驾驶舱、章节与写作、候选与正典审阅、任务中心。
- 确认高影响内容默认走 Preview / Confirm / Apply。
- 确认四个首批页面的职责边界和“工作室控制台”视觉方向。
- 新增完整 App UX 设计规格文档，用作后续实施计划和 API contract 的前置输入。

## Non-goals

- 不写业务代码。
- 不选择最终前端框架。
- 不实现账号、云端数据库、实时协作、富文本编辑器、计费或公开社区。
- 不把当前本机 Web 简单美化后宣称为最终多人在线 App。
- 不允许 Agent 或团队建议绕过作者确认直接写入正典。

## Impact

影响产品体验路线图、OpenSpec 设计记录和后续完整 App 任务拆分。该 change 只记录设计决策和验收标准，不改变 CLI/API 运行时行为。
