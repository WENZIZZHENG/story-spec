# 章节卡模板：`<章节标题>`

> 使用方式：复制到 `stories/<故事名>/chapter-cards/chapter-XXX.md` 后填写。正文写作前必须先确认本章约束卡、章节目标、冲突、信息释放、情绪变化和结尾钩子。资料不足时标为待确认，不要把 agent 推断写成正典。

## 写中沉浸原则

约束卡用于写前确认和写后自检，不作为正文生成时逐句审查器。正文阶段优先身体感、感官、动作、当下反应和句子质感；写句子时先让读者感觉角色正在经历什么，写完后再回到硬约束检查。

### 硬约束

- [ ] 情感反应没跳过。
- [ ] 能力使用符合已确认规则和代价。
- [ ] 内心独白措辞不超出当前语言水平。
- [ ] 人物行为符合当前关系和创伤反应。
- [ ] 世界观、正典和时间线不被本章改写。

### 软约束

- [ ] 人物反应有行动证据，不只靠旁白解释。
- [ ] 权力关系有刀锋感但不强行插入。
- [ ] 魔力、符文、贵族社会或其他世界规则在场景中有可见影响。
- [ ] 本章质感符合当前类型和作者已确认偏好。

## 章节小样预览

- 小样目标：`<800-1500 字左右，像缩略正文而不是纯大纲，用来确认读感、情绪顺序、人物反应、冲突推进、尺度边界和文风方向>`
- 作者反馈：`<确认 / 要求改写小样 / 补充约束 / 退回 beat>`
- 写入边界：小样默认不写入正式正文、不更新 tracking、不进入 canon。
- 扩写依据：只有作者确认或改写小样后，才进入完整章节分块生成；完整正文仍服从 Scene Card、任务边界、约束卡和写后自检。
