# 参考作品反向拆解

把作者提供的参考作品读后笔记、摘要或本地研究资料拆成原创化候选预览。

用户输入：$ARGUMENTS

## 任务边界

- 只处理作者提供的摘要、读后笔记或本地研究资料。
- 不联网抓取，不下载原作，不解析整本小说，不复述受保护原文。
- 不生成原作续写正文，不把同人续写记录误标为原创项目可发布内容。
- 不把参考作品角色名、地名、势力名、术语、剧情线或原文表达写入原创正典。
- 输出只作为 candidate / preview；进入 world、canon、specification、creative-plan 或正文前必须经过 preview / confirm / apply。

## 推荐 CLI

如当前环境可运行终端命令，优先使用：

```bash
storyspec reference:reverse $ARGUMENTS
```

如需要 JSON 交接：

```bash
storyspec reference:reverse $ARGUMENTS --json
```

## 输出结构

必须按以下分区输出：

1. 结构吸引力：作者明确喜欢、想保留或被吸引的读者体验与功能结构。
2. 原作依赖项：参考作品专有角色、势力、地名、术语、原作结局或续写意图。
3. 高风险相似项：与原作绑定过强、不能直接迁移的结构。
4. 可原创化结构：类型承诺、爽点机制、关系张力、权力结构、世界压力、未完成承诺或不适点修复。
5. 读者承诺：原创项目可以继承的阅读期待，例如解释权成长、规则可验证、关系慢热兑现或公共危机兑现。
6. 修复方向：把断更、不适点、崩坏情节或未兑现承诺转成原创问题意识。
7. 原创化指南：每个可保留结构对应的原创移动作和边界，必须去专名、去原作时间线、去受保护表达。
8. 新故事候选：去专名后的原创世界压力、主角功能、能力表达、关系承诺。
9. 不得直接照搬清单：角色名、地名、势力名、专有术语、剧情线、原文表达和未授权续写正文。

## 工作流程

1. 读取作者提供的笔记或本地 research 文件。
2. 先标出结构吸引力、原作依赖项和高风险相似项，避免喜欢点与可照搬内容混在一起。
3. 再提取可原创化结构，只保留功能、张力、承诺和读者体验。
4. 把不适点、断更或想修复的部分转成读者承诺和修复方向，不接续原作正典。
5. 输出新故事候选时替换成全新的世界压力、人物功能、能力边界和关系承诺。
6. 收尾明确：本次为预览未写入，需作者确认后才能进入正典。
